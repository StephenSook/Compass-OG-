## Findings

### F1 — Cyclomatic complexity 13 on `consumeGrantAndIssueReceipt`
**Severity:** INFO
**Detector:** [`cyclomatic-complexity`](https://github.com/crytic/slither/wiki/Detector-Documentation#cyclomatic-complexity)
**File:** `contracts/CompassHub.sol#148-203`

```text
CompassHub.consumeGrantAndIssueReceipt(CompassHub.Grant, bytes,
CompassHub.ReceiptInputs) has a high cyclomatic complexity (13).
```

**Action:** none. The high CC is the deliberate atomic-flow design —
the function performs 8 sequenced validations (provider binding, grant
expiry, policy existence + active, nullifier replay, sig length, signer
recovery + binding, receipt-fields, receipt expiry, receipt-id replay)
followed by 2 effects + 2 events, all in a single non-reentrant tx.
Splitting the function would either re-introduce a half-state grief
window or require shared storage between the split halves. The
threat-model trade is documented in `docs/honest-limits.md`; the
22-test suite at `contracts/test/CompassHub.authwit.t.ts` exercises
each branch and the property-based fuzz tests at
`contracts/test/invariants.t.ts` confirm the per-branch invariants.

### F2 — `AgentRegistry` does not implement `IAgentRegistry.ownerOf`
**Severity:** INFO
**Detector:** [`unimplemented-functions`](https://github.com/crytic/slither/wiki/Detector-Documentation#unimplemented-functions)
**File:** `contracts/AgentRegistry.sol#14-151`, `contracts/IAgentRegistry.sol#11`

```text
AgentRegistry does not implement functions:
  - IAgentRegistry.ownerOf(uint256)
```

**Action:** false positive. AgentRegistry inherits both `ERC721` and
`IAgentRegistry`, both of which declare `ownerOf(uint256) external view
returns (address)`. Solidity requires an explicit `override(ERC721,
IAgentRegistry)` to disambiguate; the override delegates to `super.ownerOf`
which resolves to ERC721's implementation:

```solidity
function ownerOf(
    uint256 tokenId
) public view override(ERC721, IAgentRegistry) returns (address) {
    return super.ownerOf(tokenId);
}
```

The compiler accepts this. The Galileo testnet deployment confirms
behavior — `agentRegistry.ownerOf(tokenId)` is called from
`CompassHub.consumeGrantAndIssueReceipt` during every receipt mint and
returns the correct owner address (verified on-chain via the
`Authwit` happy-path test in `contracts/test/CompassHub.authwit.t.ts`,
and live on Galileo at receipt-mint tx
`0xc028a63f...` per `docs/notes/0g-galileo-policy-setup.md`).

Slither's false-positive stems from its inability to trace the
`super.ownerOf` delegation across inheritance — the analyser sees the
`IAgentRegistry.ownerOf` declaration and the AgentRegistry override
separately and doesn't link them. This is a known Slither limitation,
not a bug in our contracts.

## Refactor landed in this audit

Pre-audit, the `IAgentRegistry` interface was declared inline at the
top of `CompassHub.sol`. Slither flagged this with a
`missing-inheritance` informational warning ("AgentRegistry should
inherit from IAgentRegistry"). The refactor moved the interface into
its own file `contracts/IAgentRegistry.sol`, both contracts import it,
and `AgentRegistry` now declares conformance via `is IAgentRegistry`.

The Solidity bytecode is bit-identical — interfaces don't add runtime
overhead — so the deployed Galileo contracts at
`0x461eda452ffAF43c674ef42BdccfDd6B8e13C2D8` (AgentRegistry) and
`0x60BbE5fcA6D23f7d25142E721258c641b45A7c3b` (CompassHub) need not
redeploy. Future Aristotle mainnet deploy uses the cleaned source.
